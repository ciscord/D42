{% trans "Mergers and Acquisitions" %}

{% trans 'Challenge' %}

{% trans "Mergers and acquisitions are a lot more than paperwork and a name change. When companies merge, each company will initially have its own supporting IT Infrastructure that comes along with it. There will likely be a lot of duplication of both hardware and services between the two infrastructures, with each company relying on the services from their own Active Directory domains, e-mail servers & services, and web hosting arrangements, just to name a few." %}

{% trans "Now that the companies are merging, their data-centers need to merge as well. It needs to be decided what services from each infrastructure are critical, and will remain, and which will be consolidated or eliminated. There may be two different answers for both the short term and long term, as well. Because of this, the process involves in depth planning, testing, and careful execution, as neither party was likely at all familiar with the others infrastructure before the merger or acquisition. There is also can exist much risk and many unknowns when merging AD forests and domain trusts, for example, as this grants various access rights across unfamiliar domains." %}

{% trans "You need agility and visibility to smoothly handle mergers and to consolidate assets from acquisitions. Whether your goal is to migrate one or both infrastructures to the public cloud, to a private cloud platform like OpenStack or Kubernetes, to combine the two into one of the existing data-centers, or to migrate everything to an entirely new facility, you need to be able to answer questions such as" %}:

  • {% trans "Where are the existing physical, virtual, and cloud devices for each infrastructure" %}
  • {% trans "How are the devices currently networked (VLANs, IP blocks) within each data center" %}
  • {% trans "Routing & connectivity between the facilities before, during, and after cutover" %}
  • {% trans "Will there be IP re-use, or will IP space be renumbered, and are enough IPs available" %}
  • {% trans "What storage components are currently in use and their underlying connectivity" %}
  • {% trans "What services and software is running and what specifically needs to move" %}
  • {% trans "What is running on all the machines to be migrated, and their interdependencies" %}
  • {% trans "What users depend on services that will be migrated and notification if necessary" %}
  • {% trans "Where are all the credentials (passwords, SSH Keys, etc.)" %}
  • {% trans "Are all configurations and config files known and correct for the new facility" %}
  • {% trans "Licensing for all software in use at each facility" %}

{% trans "If migrating to another physical data-center, as well as the above, you must also consider" %}:

  • {% trans "Power supply to the facility (both utility & generator availability)" %}
  • {% trans "Power wiring for redundancy and circuit balancing" %}
  • {% trans "Network wiring" %}
  • {% trans "Tracking vendor service and warranty contracts" %}
  • {% trans "Logistics regarding equipment in transit from one data center to the new facility" %}
  • {% trans "Determination of what will be kept, retired, replaced, or consolidated" %}
