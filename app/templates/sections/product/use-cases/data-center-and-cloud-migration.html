{% trans "Data Center & Cloud Migration" %}

{% trans 'Challenge' %}

{% trans "Data Center migrations require extremely careful planning and execution. They also carry significant business risk, as botched cutovers can results in runaway costs compensating customers for downtime, as well as unplanned spend on extra time in the original data center." %}

{% trans "Both agility and clear visibility are required to execute a successful data center migration. Migrations are typically very stressful times for the IT Staff involved, as the weight of a successful cutover that mitigates the aforementioned risk rests on their shoulders. Detailed knowledge of your infrastructure and a clear picture of the inter-dependencies between the many layers of hardware, supporting infrastructure, and the business applications they support are critical to the undertaking." %}

{% trans "Whether your goal is to migrate to a new hosting facility, to the public cloud, to a private cloud platform like OpenStack, or from one cloud vendor to another, you need to be able to answer questions such as:" %}

  • {% trans "Where are my existing physical, virtual, and cloud devices?" %}
  • {% trans "How are these devices currently networked (VLANs, IP blocks)?" %}
  • {% trans "Will the new space re-use numbering, or be re-IP'd and are those IPs available?" %}
  • {% trans "Routing & connectivity between the facilities before, during, and after cutover" %}
  • {% trans "What storage components are currently in use and their underlying connectivity?" %}
  • {% trans "What services and software is running and what specifically needs to move" %}
  • {% trans "What is running on all the machines to be migrated, and their interdependencies" %}
  • {% trans "What users depend on services that will be migrated and notification if necessary" %}
  • {% trans "Where are all the credentials (passwords, SSH Keys, etc.)" %}
  • {% trans "What licenses are in use by each server, and is there a shortage or duplicates?" %}
  • {% trans "Are all configurations and config files known and correct for the new facility" %}

{% trans "If migrating to another physical data-center, as well as the above, you must also consider" %}:

  • {% trans "Power supply to the facility (both utility & generator availability)" %}
  • {% trans "Power wiring for redundancy and circuit balancing" %}
  • {% trans "Network wiring" %}
  • {% trans "Tracking vendor service and warranty contracts" %}
  • {% trans "Logistics regarding equipment in transit from one data center to the new facility" %}
  • {% trans "Determination of what will be kept, retired, replaced, or consolidated" %}
