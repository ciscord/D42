{% trans "Link CMDB items to issues in JIRA" %}

{% trans "Once the Device42 configuration management database is synced to JIRA, users can select any of the CIs to create issues in JIRA. Currently the sync encompasses devices, power units, parts and racks. When new issues are created in JIRA, users can select multiple CIs from a drop-down list or search for CIs by name to refine the list selections. User can add one or more Device42 custom fields as well." %}

{% trans "View CI details from JIRA helpdesk issues" %}

{% trans "Once the Device42 JIRA connection is made, users can see details from Device42 (such as serial #, asset #, and hardware model) for that CI in JIRA. Similarly, you can see the associated info for other CI types. Conversely, the CIs in JIRA can be linked to view the Device42 page for that Configuration Item enabling users to easily view configuration item details in Device42." %}

{% trans "CIs from the issue navigator" %}

{% trans "If you add the Device42 custom field to the issue navigator, you will be able to see brief information about each CI that is linked to the ticket with the ability to follow the link to jump into the Device42 application. If you hover over the link you will see detailed information about the device." %}

{% trans "Search JIRA service requests by Configuration items" %}

{% trans "Once the CIs from the Device42 IT Asset Management system are synced in JIRA, you can do simple search or advanced search based on JQL (JIRA Query Language). For simple search, you can go to “more” in the issue navigator and choose the Device42 custom field to query. For Advanced search, you can go to Advanced search mode and search using JQL." %}
